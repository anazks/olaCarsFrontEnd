# Frontend Integration Guide: Agreement Module

This guide provides everything you need to know to integrate the newly built **Agreement Module** on the frontend, specifically tailored for TipTap editors.

## Overview
The Agreement module stores versioned legal/system documents like "Terms and Conditions", "Privacy Policy", etc. The backend directly accepts the HTML string generated by TipTap and automatically sanitizes it to prevent XSS.

**Base URL**: `/api/agreements`

**Allowed Roles for modifications**: `ADMIN`, `OPERATIONADMIN`, `COUNTRYMANAGER`

---  

## 1. Create a New Agreement
Used to save a brand new agreement document.

- **Endpoint**: `POST /api/agreements`
- **Auth headers needed**: Yes (`Authorization: Bearer <token>`)

### Request Body
```json
{
  "title": "Terms and Conditions",
  "country": "US",
  "type": "TERMS_AND_CONDITIONS", 
  "content": "<h1>Terms Use</h1><p>Welcome to our platform...</p>",
  "status": "PUBLISHED"
}
```
*Note: `title` must be unique per `country`.*
*Note for `type`: Values can be "TERMS_AND_CONDITIONS", "PRIVACY_POLICY", "RETURN_POLICY", "OTHER".*

### Success Response (201 Created)
```json
{
  "status": "success",
  "message": "Agreement created successfully",
  "data": {
    "_id": "64abcdef123... ",
    "title": "Terms and Conditions",
    "country": "US",
    "type": "TERMS_AND_CONDITIONS",
    "content": "<h1>Terms Use</h1><p>Welcome to our platform...</p>",
    "version": 1,
    "status": "PUBLISHED",
    "createdBy": "123user_id...",
    "creatorRole": "ADMIN"
  }
}
```

## 2. Edit an Existing Agreement
Updates an existing agreement. If `content`, `country`, `title`, or `type` is changed, the backend automatically increments the `version` number (e.g., from 1 to 2) and saves the previous state to the `AgreementVersion` history collection.

- **Endpoint**: `PUT /api/agreements/:agreementId`
- **Auth headers needed**: Yes

### Request Body
Pass the new HTML content extracted from your TipTap editor:
```json
{
  "content": "<h1>Updated Terms</h1><p>We changed our policies...</p>"
}
```
*(You can also pass `title`, `country`, `type`, or `status` to modify them).*

### Success Response (200 OK)
```json
{
  "status": "success",
  "message": "Agreement updated successfully",
  "data": {
    "_id": "64abcdef123...",
    "version": 2,
    "content": "<h1>Updated Terms</h1><p>We changed our policies...</p>"
    // ...other fields
  }
}
```

---

## 3. Get All Agreements
Fetches the latest version of all agreements active in the system.

- **Endpoint**: `GET /api/agreements`
- **Query Params**: `?country=US` (Filter by country)
- **Auth headers needed**: No (Public) // *Change if frontend wants this hidden*

### Success Response (200 OK)
```json
{
  "status": "success",
  "data": [
    {
      "_id": "64abcdef123...",
      "title": "Terms and Conditions",
      "country": "US",
      "type": "TERMS_AND_CONDITIONS",
      "version": 2,
      "status": "PUBLISHED"
      // ... content is also returned
    }
  ]
}
```

---

## 4. Get a Specific Agreement
Useful for displaying an agreement on its dedicated page.

- **Endpoint**: `GET /api/agreements/:agreementId`
- **Auth headers needed**: No (Public)

---

## 5. View Agreement History (Versions)
Fetches all past versions of a specific agreement document. Useful if the admin wishes to see how the agreement evolved over time.

- **Endpoint**: `GET /api/agreements/:agreementId/versions`
- **Auth headers needed**: Yes

### Success Response (200 OK)
```json
{
  "status": "success",
  "data": [
    {
      "agreementId": "64abcdef123...",
      "version": 2,
      "country": "US",
      "content": "<h1>Updated Terms</h1>...",

      "agreementId": "64abcdef123...",
      "version": 1,
      "country": "US",
      "content": "<h1>Terms Use</h1>...",
      "updatedBy": "123admin_id..."
    }
  ]
}
```

## TipTap Frontend Usage Advice

**1. Getting HTML directly from TipTap:**
When the admin hits "Save", retrieve the HTML from the editor instance using:
```javascript
const htmlContent = editor.getHTML();
// Send this htmlContent in the 'content' field via Axios/Fetch API
```

**2. Hydrating TipTap for Editing:**
When the admin clicks "Edit", fetch the agreement data via `GET /api/agreements/:id`. Load the HTML string natively into the TipTap initialization setup or state:
```javascript
const editor = useEditor({
  extensions: [...],
  content: backendAgreementData.content, // Load existing string directly
})
```

**Security note**: The backend already runs `sanitize-html` to filter out `<script>` injections or unwanted iframe tags. Your generated frontend styles inline via TipTap will be preserved (except risky attributes).
